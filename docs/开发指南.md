# 开发指南

## Git工作流
1. `git pull origin main` - 每天开始前
2. `git checkout -b feature/名字-功能` - 创建分支
3. `git add . && git commit -m "feat: 描述"` - 提交
4. `git push origin feature/名字-功能` - 推送
5. 在GitHub创建Pull Request

## 代码规范
- 遵循PEP8
- 函数和类写docstring
- 变量用英文名
- 注释用中文

## 提交信息格式
- `feat:` 新功能
- `fix:` 修复bug
- `docs:` 文档
- `style:` 格式
- `refactor:` 重构
